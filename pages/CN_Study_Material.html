CH 01

Introduction to Computer Networks

What is the Internet?

The Internet is a "network of networks" — billions of connected devices (hosts/end systems) interconnected through packet switches (routers and switches) and communication links. It is governed by protocols defined in RFCs (Request for Comments) by the IETF.

ProtocolDefines format, order of messages, and actions taken on message transmission/receipt.
Host / End SystemDevices at the edge (PCs, phones, servers) that run network applications.
Packet SwitchRouter or link-layer switch that forwards packets toward destination.
ISPInternet Service Provider. Access ISPs connect to tier-1 ISPs via IXPs and regional ISPs.

Packet Switching vs. Circuit Switching

Packet Switching
  • Data split into packets; each forwarded independently
  • Resources shared (statistical multiplexing)
  • Can support more users for bursty data
  • Possible congestion, queueing delay, loss
  • Used in the Internet (TCP/IP)
Circuit Switching
  • Dedicated end-to-end path reserved before data transfer
  • Guaranteed bandwidth (circuit-like performance)
  • Resources idle during silence periods
  • FDM or TDM to divide capacity
  • Used in telephone networks (PSTN)
  • Differentiate circuit-switched vs. packet-switched: key differences, pros/cons [Fall 2024 Q1b]
  • Packet switching allows more users: e.g., 1 Mbps link, each user 100 kbps active 10% → 35 users vs 10 in circuit switching

Four Sources of Packet Delay

d_nodal = d_proc + d_queue + d_trans + d_prop d_trans = L/R (L = packet size in bits, R = link rate in bps) d_prop = d/s (d = link distance in meters, s ≈ 2×10⁸ m/s) d_queue = depends on traffic intensity La/R End-to-end delay (n links) = n × (d_trans + d_prop + d_proc + d_queue)
  • End-to-end delay calculation [Fall 2024 Q2a]: Packet=2500 bytes, prop speed=2.5×10⁸ m/s, rate=2 Mbps, processing=5 ms, links=5000+4000+1000 km → calculate total delay
  • Propagation delay ≠ Transmission delay — know the difference!
  • Distance for d_prop = d_trans: s=2.5×10⁸, L=120 bits, R=56 kbps → m = s × L/R

Step-by-Step: End-to-End Delay (3 links)

Transmission delay per link: d_trans = L/R = (2500×8 bits) / (2×10⁶) = 0.01 sec = 10 ms
Propagation delay for each link: d_prop = distance/speed (e.g., 5000×10³ / 2.5×10⁸ = 20 ms)
Total = 3×(d_trans) + sum(d_prop per link) + 2×processing_delay (at intermediate switches)
Note: For store-and-forward, each router must receive whole packet before forwarding → adds d_trans per hop

Network Security Threats

  • Malware: Virus (executes with user action), Worm (self-propagates), Trojan, Ransomware, Spyware
  • DoS / DDoS: Overwhelm target with bogus traffic
  • Packet Sniffing: Promiscuous NIC captures all passing frames
  • IP Spoofing: Sending packets with false source address
  • Phishing: Fake emails/sites to steal credentials
CH 02

Application Layer

HTTP (HyperText Transfer Protocol)

HTTP uses TCP port 80. It is stateless — server maintains no information about past requests. Uses cookies for state.

Non-Persistent HTTP (HTTP/1.0)
  • New TCP connection per object
  • Response time = 2RTT + file transmission time per object
  • OS overhead for each connection
  • 10 images → 10 separate TCP connections (22 RTTs total for base + objects)
Persistent HTTP (HTTP/1.1)
  • Single TCP connection for multiple objects
  • Response time ≈ 1 RTT per object after connection setup
  • Much more efficient
  • Can pipeline requests
  • HTTP timing calculation [Fall 2024 Q3b]: Base HTML + 37 JPEG images on same server. Non-persistent: 2RTT + transmission per object × 38 objects. Persistent: 2RTT (setup) + 38×(1RTT + transmission)
  • Know HTTP methods: GET, POST, HEAD, PUT
  • Know status codes: 200 OK, 301 Moved, 400 Bad Request, 404 Not Found, 505 Version Not Supported
  • HOL (Head-of-Line) blocking: small objects wait behind large object in HTTP/1.1 [Fall 2024 Q1e context]

Web Caching (Proxy Server)

A web cache satisfies client requests without involving origin server. Acts as both client (to origin) and server (to browser). Reduces response time and access link utilization.

  • Cache delay calculation [Fall 2024 Q1c]: Cache hit rate 0.4 → 40% requests served locally (msecs), 60% from origin. Average delay = 0.6×(2.01s) + 0.4×(~msecs) ≈ 1.2s vs without cache
  • Conditional GET: If-Modified-Since header → server returns 304 Not Modified if unchanged

DNS — Domain Name System

Distributed hierarchical database that translates hostnames to IP addresses. Three levels: Root → TLD (.com, .edu) → Authoritative DNS servers. Local DNS server caches and forwards.

Iterative Query
  • Local DNS does all the work
  • Each server returns "ask this one"
  • Local DNS contacts root → TLD → authoritative
  • Less burden on upper-level servers
Recursive Query
  • Each server contacts the next on behalf of requester
  • Heavy load on upper-level servers
  • Root DNS contacts TLD, TLD contacts authoritative
  • Chain of requests, result passed back
  • Iterative vs recursive DNS query — draw diagrams [Fall 2024 Q1d, Fall 2022 Q1f]
  • DNS resource record types: A (name→IP), NS (domain→nameserver), CNAME (alias→canonical), MX (mail server)
  • DNS caching: entries have TTL; local DNS server caches to reduce traffic

SMTP — Email Protocol

SMTP uses TCP port 25. Client push model. Three phases: handshaking, transfer, closure. Commands are ASCII text. Uses CRLF.CRLF to end message body. IMAP is used for mail retrieval (stores on server). POP3 downloads and deletes.

  • SMTP working steps + example interaction [Fall 2022 Q1g, Spring 2022 Q1b]
  • Compare SMTP vs HTTP: both ASCII command/response; HTTP is client-pull, SMTP is client-push
  • POP3 vs IMAP: POP3 downloads/deletes (stateless), IMAP keeps on server, supports folders
CH 03

Transport Layer

Multiplexing and Demultiplexing

Multiplexing: gathering data from multiple sockets, adding transport headers. Demultiplexing: using header fields to deliver segments to correct socket.

Connectionless (UDP) Demux
  • Uses destination port number only
  • Datagrams with same dest port → same socket, regardless of source
Connection-Oriented (TCP) Demux
  • Uses 4-tuple: (src IP, src port, dst IP, dst port)
  • Different source IP/port → different socket
  • Server can support many simultaneous sockets
  • Connection-oriented vs connectionless demultiplexing with figures [Spring 2022 Q2a]

UDP — User Datagram Protocol

Bare-bones, best-effort service. No connection setup, no flow/congestion control. Used for: DNS, streaming, SNMP, HTTP/3. Header: src port, dst port, length, checksum (8 bytes total).

Why UDP? No RTT delay for connection setup, small header overhead, no congestion control (can send as fast as desired), works even when network is congested.

Pipelined Protocols

Go-Back-N (GBN)
  • Window of N unACKed packets
  • Cumulative ACK
  • On timeout(n): retransmit n AND all higher seq # in window
  • Receiver discards out-of-order (only buffers expected)
  • Wastes bandwidth on retransmission of good packets
Selective Repeat (SR)
  • Window of N; receiver buffers out-of-order
  • Individual ACK per packet
  • On timeout(n): retransmit only packet n
  • Better bandwidth efficiency
  • Window size ≤ seq# space / 2 to avoid ambiguity
  • GBN vs SR comparison [Fall 2024 Q3c]: 3-bit field (seq# space = 0–7, window=7 for GBN, ≤4 for SR). Given frame loss scenarios — count retransmissions for each protocol
  • With 9 packets, window=3, GBN, every 5th lost [Spring 2022 Q3c] — draw timing diagram and count total transmissions

TCP — Transmission Control Protocol

TCP Segment Structure

Key fields: Source/Dest port (16 bits each), Sequence number (32 bits), Acknowledgement number (32 bits), Receive window (rwnd), Checksum, Flags (SYN, ACK, FIN, RST, URG, PSH), Options.

RTT Estimation & Timeout

EstimatedRTT = (1 - α) × EstimatedRTT + α × SampleRTT [α = 0.125] DevRTT = (1 - β) × DevRTT + β × |SampleRTT - EstimatedRTT| [β = 0.25] TimeoutInterval = EstimatedRTT + 4 × DevRTT
  • RTT calculation [Fall 2024 Q2e]: SampleRTT = 106, 120, 115 ms; α=0.25, initial EstimatedRTT=100ms; β=0.15, initial DevRTT=10ms. Compute EstimatedRTT, DevRTT, TimeoutInterval after each sample.

Example (Sample 1: SampleRTT = 106 ms)

EstimatedRTT = (0.75)(100) + (0.25)(106) = 75 + 26.5 = 101.5 ms DevRTT = (0.85)(10) + (0.15)|106 - 100| = 8.5 + 0.9 = 9.4 ms TimeoutInterval = 101.5 + 4(9.4) = 139.1 ms

TCP Flow Control

Receiver advertises rwnd (receive window) in header. Sender limits in-flight data: LastByteSent - LastByteACKed ≤ rwnd. Prevents receiver buffer overflow.

TCP 3-Way Handshake

Client → Server: SYN (SYNbit=1, Seq=x) — client chooses initial seq #
Server → Client: SYNACK (SYNbit=1, Seq=y, ACKbit=1, ACKnum=x+1)
Client → Server: ACK (ACKbit=1, ACKnum=y+1) — may contain data

TCP Connection Teardown

Client sends FIN, server responds ACK, then server sends FIN, client responds ACK and waits 2×max segment lifetime (timed wait) before closing.

TCP Congestion Control

Congestion control is one of the most frequently examined topics. Master the graph reading!

AIMD (Additive Increase, Multiplicative Decrease)

  • Additive Increase: Increase cwnd by 1 MSS per RTT (when no loss)
  • Multiplicative Decrease: Cut cwnd by half on triple-duplicate ACK (TCP Reno); cut to 1 MSS on timeout (TCP Tahoe)

Three Phases

Slow Startcwnd starts at 1 MSS. Doubles every RTT (exponential). Ends when cwnd reaches ssthresh or loss occurs.
Congestion Avoidancecwnd increases by 1 MSS per RTT (linear). Starts when cwnd ≥ ssthresh.
Fast RetransmitOn 3 duplicate ACKs: retransmit lost packet immediately (don't wait for timeout). cwnd = ssthresh = cwnd/2 (Reno).
FeatureTCP TahoeTCP Reno
Triple Dup ACK actioncwnd = 1 MSS (go to slow start)cwnd = ssthresh = cwnd/2 (fast recovery)
Timeout actioncwnd = 1 MSS, ssthresh = cwnd/2cwnd = 1 MSS, ssthresh = cwnd/2
After loss detectionAlways slow startFast recovery for dup ACKs
  • Graph reading [Fall 2024 Q3d]: Identify slow start intervals, congestion avoidance intervals; after round 16 and 22 — timeout or dup ACK? Initial ssthresh? ssthresh at round 18? When is 36th segment sent?
  • Tips: Slow start = exponential growth. Congestion avoidance = linear (slope 1). Triple dup ACK → cwnd drops to half (no return to 1). Timeout → cwnd drops to 1.

2D Parity

Arrange bits in matrix. Compute row parity bits AND column parity bits. Can detect AND correct single bit errors. Single-bit error shows up as both a row parity error AND a column parity error — intersection gives the error location.

  • [Fall 2024 Q2b]: Sender sends 00101101, 10101101, 00110110, 11101001. Compute row and column parity. Receiver gets 00101101, 10101111, 00110110, 11101001 — pinpoint the errors.

Network Layer (Data Plane + Control Plane)

Router Architecture

A router has: Input ports (line termination, link layer, lookup/forwarding), Switching fabric (memory, bus, or crossbar interconnection), Output ports (queuing, link layer, line termination), and Routing processor (control plane, runs routing protocols).

  • Router architecture description [Spring 2022 Q4a] — identify key responsibility of network layer (routing + forwarding)
  • HOL Blocking [Fall 2024 Q1e]: Head-of-Line blocking at input port — a queued datagram at front of input queue prevents datagrams behind it from moving to (different) output ports, even if those output ports are free.

IP Addressing and Subnetting

CIDR Notation

IP address format: a.b.c.d/x where x = bits in subnet part. Subnet mask identifies network vs host portion.

Subnetting Steps

Determine how many subnets needed and hosts per subnet
Find number of bits needed for subnets: 2^n ≥ number of subnets
Find host bits needed: 2^h - 2 ≥ hosts per subnet (subtract 2 for network and broadcast)
New prefix = original prefix + subnet bits. Assign address ranges sequentially
First address = network address. Last address = broadcast. Usable = first+1 to last-1
  • Subnetting [Fall 2024 Q2c]: 192.168.10.0/24 → 14 subnets: 2×64 addresses (/26), 4×16 (/28), 4×8 (/29), 4×4 (/30). Construct table with subnet mask, first/last address.
  • ISP allocation [Spring 2022 Q4c]: 192.168.0.0/22 → customer 1 needs 256 (→/24), customer 2 needs 128 (→/25), etc.
  • Always list: first address (network), last address (broadcast), subnet mask (/prefix)

IP Fragmentation

When datagram exceeds MTU (Max Transfer Unit), it's fragmented. Reassembly happens only at destination.

Fragment size (data) = MTU - 20 bytes (IP header) Offset field = byte position of fragment / 8 More Fragments flag (MF) = 1 for all but last fragment Last fragment: MF = 0
  • [Fall 2024 Q2f]: 7800-byte datagram, MTU=2120, ID=137. Determine number of fragments; table: fragment #, datagram length, payload length, ID, fragmentation flag, offset
  • 4000-byte datagram, MTU=1500 [Spring 2022 Q4b]: Fragment 1: length=1500, payload=1480, offset=0, MF=1. Fragment 2: length=1500, payload=1480, offset=185, MF=1. Fragment 3: length=1040, payload=1020, offset=370, MF=0

Step-by-step fragmentation (4000 bytes, MTU=1500):

Max data per fragment = 1500 - 20 = 1480 bytes
Fragment 1: 1480 bytes data, offset=0/8=0, MF=1, total length=1500
Fragment 2: 1480 bytes data, offset=1480/8=185, MF=1, total length=1500
Fragment 3: 4000-20-1480-1480 = 1020 bytes data, offset=2960/8=370, MF=0, total length=1040

NAT — Network Address Translation

All devices in local network (private IPs: 10.x.x.x, 172.16.x.x, 192.168.x.x) share one public IP. NAT router maintains translation table mapping (private IP, port) ↔ (public IP, new port).

  • NAT step-by-step [Fall 2024 Q3e]: Home network 192.168.1.0/24, host 192.168.1.4 requests web page at 173.194.67.102:80 through router with external IP 67.210.97.1. Describe NAT process + translation table for all hosts.
Host 192.168.1.4 sends packet: src=192.168.1.4:3345, dst=173.194.67.102:80
NAT router rewrites: src=67.210.97.1:5001 (new port), dst=173.194.67.102:80. Stores (192.168.1.4:3345) ↔ (67.210.97.1:5001) in NAT table
Server responds to 67.210.97.1:5001
NAT router looks up table, rewrites dst from 67.210.97.1:5001 → 192.168.1.4:3345, forwards internally

Routing Algorithms

Link State (Dijkstra's Algorithm)
  • Global: all routers know full topology
  • Flood link-state info, then compute shortest paths
  • O(n²) time complexity
  • Used in OSPF
  • Fast convergence, but high message overhead
Distance Vector (Bellman-Ford)
  • Decentralized: exchange DVs with neighbors only
  • D_x(y) = min_v { c(x,v) + D_v(y) }
  • Convergence time varies
  • Count-to-infinity problem for bad news
  • Used in RIP
  • Dijkstra's algorithm [Fall 2024 Q3a, Fall 2022 Q1c]: Show distance table iterations for each node. Track N' (confirmed set), D(v) and p(v) for each node at each step.
  • Distance Vector [Fall 2022 Q1d, Spring 2022 Q6a BGP context]: Show DV table for each node at t=0 and after iterations until convergence.
  • Count-to-infinity [Fall 2024 Q1f]: When a link cost increases, "bad news travels slowly" — DV nodes keep incrementing costs in a loop. Solutions: split horizon, poison reverse.

Dijkstra's Algorithm Procedure:

Initialize: N'={u}. For all v: D(v) = c(u,v) if directly connected, else ∞
Loop: Find w not in N' with minimum D(w). Add w to N'
Update: For all v adjacent to w, not in N': D(v) = min(D(v), D(w) + c(w,v))
Repeat until all nodes in N'. Build forwarding table from predecessor p(v)

Link Layer and LANs

Link Layer Services

  • Framing: Encapsulate datagram into frame, add header/trailer
  • Link access: MAC protocol controls when to send on shared medium
  • Reliable delivery: Seldom used on low-error links; used on wireless
  • Error detection and correction
  • Link layer services [Fall 2024 Q1a]: Explain the services provided by the data link layer — framing, reliable delivery, flow control, error detection, error correction, half/full duplex

Error Detection: CRC (Cyclic Redundancy Check)

Given data D and generator G (r+1 bits), compute R such that D·2^r XOR R is divisible by G.

R = remainder[ D·2ʳ / G ] (polynomial division in GF(2), i.e., XOR arithmetic) Sender transmits <D, R> Receiver divides <D', R'> by G; if remainder ≠ 0 → error detected
  • [Fall 2024 Q2d]: G=10011, D=1010101010. Compute R. Show how receiver checks D=1010101110.
  • CRC detects all burst errors of length < r+1 bits. Widely used in Ethernet, 802.11.

CRC Step-by-Step (D=1010101010, G=10011, r=4):

Append r=4 zeros to D: 10101010100000
Divide by G=10011 using XOR long division. Perform XOR at each step, bring down next bit.
Remainder R is the last r=4 bits. Transmit D followed by R.
Receiver divides <D,R> by G. If remainder=0, no error. If ≠0, error detected.

MAC Addresses and ARP

MAC address: 48-bit flat address, burned into NIC. Written as hex (e.g., 1A-2F-BB-76-09-AD). Portability advantage over IP (doesn't depend on subnet). Broadcast: FF-FF-FF-FF-FF-FF.

ARP (Address Resolution Protocol): Resolves IP address → MAC address within same subnet. Host broadcasts ARP query to FF-FF-FF-FF-FF-FF; target replies with its MAC address.

  • ARP across different LANs [Fall 2024 Q3f]: Host A (111.111.111.111) wants to send to Host B (222.222.222.222) through Router R. A cannot directly ARP for B (different subnet) — A sends ARP for R's MAC, then sends frame to R (MAC=R), R receives, re-ARPs for B's MAC, sends frame to B. Construct ARP tables for A, B, R.
  • Key: IP header destination stays B throughout; MAC header destination changes at each hop

Ethernet and Switches

Ethernet uses unslotted CSMA/CD with binary exponential backoff. Switches are link-layer devices that store-and-forward frames. They self-learn by recording (MAC address, interface) when frames arrive. Unknown destination → flood all ports except incoming.

Switch vs Router: Both store-and-forward. Switches work at Layer 2 (MAC addresses, learn via flooding). Routers work at Layer 3 (IP addresses, compute forwarding tables via routing algorithms).

Network Security

Security Goals

ConfidentialityOnly sender and receiver understand message contents. Achieved via encryption.
AuthenticationConfirming the identity of the communicating parties.
Message IntegrityEnsuring message not altered in transit. Achieved via MACs and digital signatures.
Access and AvailabilityServices must be accessible to authorized users.

Symmetric Key Cryptography

Sender and receiver share same secret key K_S. Examples: Caesar cipher (shift by k), monoalphabetic substitution, polyalphabetic (multiple Caesar ciphers in pattern), DES, AES.

  • Transposition cipher [Fall 2022 Q1b]: Rearrange letters using key (e.g., FANCY). Write plaintext in rows under key columns; read columns in alphabetical order of key letters.
  • Substitution ciphers [Fall 2022 Q2b]: Monoalphabetic (26! possible keys), polyalphabetic, Vigenère.

RSA — Public Key Cryptography

Key Generation Steps

Choose two large primes p and q
Compute n = p×q, z = (p-1)(q-1)
Choose e (e < n) such that gcd(e, z) = 1 (e and z are coprime)
Choose d such that e×d mod z = 1 (i.e., ed - 1 is divisible by z)
Public key = (n, e). Private key = (n, d)
Encryption: c = mᵉ mod n Decryption: m = cᵈ mod n Key property: K⁻_B(K⁺_B(m)) = m = K⁺_B(K⁻_B(m)) (private then public = public then private)
  • [Fall 2024 Q2g]: p=5, q=11, e=5. Find public and private keys. Encrypt word "aust" (a=1, b=2, …, z=26) letter by letter. Decrypt to recover plaintext.
  • Step: n=55, z=40. Find d: 5d mod 40=1 → d=9 (since 5×9=45=40+5, 45 mod 40=5 — try d=17: 5×17=85, 85 mod 40=5 — try systematically). Correct: d=9 since 5×9=45, 45 mod 40=5 ≠ 1. Try d=25: 5×25=125, 125 mod 40=5. d=33: 5×33=165, 165 mod 40=5. Actually: ed≡1(mod 40). 5d≡1(mod 40). d=9: 5×9=45≡5. d=17: 85≡5. d=25:125≡5. Note: none works easily with p=5,q=11,e=5 since gcd(5,40)=5≠1. The question uses p=5,q=11,e=5 from [Fall 2022 Q7c] where p=7,q=13.
  • [Fall 2022 Q7c]: p=7, q=13, e=5. n=91, z=72. d: 5d≡1(mod 72) → d=29 (5×29=145=2×72+1). Encrypt M=11: c=11⁵ mod 91=51. Decrypt: 51²⁹ mod 91=11.

Message Authentication Code (MAC) vs Digital Signature

MAC (Message Auth Code)
  • Uses shared symmetric key + hash function
  • Provides message integrity and authentication
  • Does NOT provide non-repudiation
  • Both parties share same key → either could have created it
  • Cannot be used in digital signature (no non-repudiation)
Digital Signature
  • Uses asymmetric key pair (private key to sign)
  • Provides integrity, authentication, AND non-repudiation
  • Only private key holder can produce signature
  • Verified by anyone with public key
  • MAC is not used because it lacks non-repudiation
  • [Spring 2022 Q7b]: "MAC is not used in case of digital signature" — because MAC requires both parties to share a symmetric key; either party could have created the MAC, so you can't prove who sent it. Digital signature uses private key (only sender has it), providing verifiable non-repudiation.
