CH 01

Introduction to AI and Rational Agents

What is Artificial Intelligence?

AI attempts to understand and build intelligent entities. Four major approaches historically distinguish different schools of thought:

ApproachFocusMeasure of Success
Act like humansTuring Test — can a machine be indistinguishable from a human?Fooling human interrogators
Think like humansCognitive science — model human thought processes experimentallyMatch human reasoning patterns
Think rationallyLaws of thought — implement logic and correct inferenceLogically valid conclusions
Act rationallyRational agent — take best possible action given knowledgeExpected performance maximized

The modern approach focuses on rational agents — systems that act to achieve the best expected outcome given available information. This is the framework used in this course.

AI draws from many fields: Philosophy (what is thinking?), Mathematics (logic, probability), Economics (utility, decisions), Neuroscience (biological neurons), Psychology (human behavior), Linguistics (language), Computer Engineering, and Control Theory.

Historical Milestones

  • 1940s–50s: Simple circuit models of neurons; McCulloch and Pitts
  • 1956: Official birth — Dartmouth Workshop, John McCarthy coins "Artificial Intelligence"
  • 1980s: Knowledge-based expert systems (MYCIN, XCON); then stagnation ("AI winter")
  • 1986+: Neural network renaissance (backpropagation); probabilistic reasoning
  • 1995+: Intelligent agent paradigm; Internet scale AI
  • 1997: IBM Deep Blue defeats Kasparov at chess
  • 2001+: Learning from large datasets; statistical ML mainstream
  • 2011+: Deep learning revolution; ImageNet, GPT, AlphaGo, LLMs
  • 2017: AlphaGo Zero learns Go from scratch, defeats all humans

Rational Agents

An agent is anything that perceives its environment through sensors and acts upon it through actuators. Humans use eyes/ears (sensors) and hands/voice (actuators). Software agents use files and network packets.

A rational agent chooses actions that maximize expected performance given its percept sequence and knowledge base. It also learns — improves based on experience — and functions autonomously.

Four Basic Agent Types

Simple Reflex AgentActs only on current percept. Uses condition-action rules (if dirty → suck). No memory. Works only in fully observable environments.
Model-Based Reflex AgentMaintains internal state (model of world) to handle partial observability. Updates state based on history of percepts. Still rule-based.
Goal-Based AgentHas explicit goal(s). Searches or plans to find action sequences that achieve goals. More flexible — can handle new goals without reprogramming.
Utility-Based AgentMaximizes an explicit utility function (measure of how "happy" the agent is). Handles tradeoffs between competing goals. Most powerful and general.

PEAS — Task Environment Specification

To design a rational agent, specify its PEAS:

P — Performance MeasureHow is success measured? E.g., score in game, distance saved, customer satisfaction.
E — EnvironmentWhat is the agent operating in? E.g., road, game board, hospital ward.
A — ActuatorsHow does the agent act? E.g., steering wheel, keyboard, voice.
S — SensorsHow does the agent perceive? E.g., camera, keyboard input, GPS.

Environment Properties

PropertyOptionsExplanation
ObservabilityFully / Partially ObservableCan agent see complete state? Chess = fully; driving = partially (hidden road ahead)
DeterminismDeterministic / StochasticDoes action fully determine next state? Deterministic = yes; stochastic = has randomness
EpisodesEpisodic / SequentialDoes current action affect future? Episodic = no (classify each image); sequential = yes (chess)
DynamicsStatic / DynamicDoes environment change while agent thinks? Taxi driving = dynamic; crossword = static

CH 02

Knowledge Representation and Reasoning

Propositional Logic (PL)

Propositional Logic is a formal language for representing knowledge (facts and rules) in a Knowledge Base (KB) and deriving new facts by inference. KB contains definite Horn clauses — clauses with exactly one positive literal. A rule like A∧B→C is represented as a Horn clause.

Inference means deriving new sentences from existing ones. Two key algorithms are used when the KB contains only Horn clauses:

Forward Chaining (Data-Driven)

Start from known facts, apply rules, derive new facts, repeat until query is found or no new facts can be derived. Called data-driven because it starts from data and moves toward the goal.

Process: Known facts → match premises of rules → derive conclusions → add to KB → repeat Query: KB ⊨ M? Example KB: A, B, C, D, P∧Q, C∧L→P, D∧M→P, B∧L→M, A∧P→L, A∧B→L, A∧D→G, G∧B→C Step 1: M not in KB Step 2: A∧B → L (A and B both known) → L added Step 3: A∧D → G (A and D known) → G added Step 4: B∧L → M (B and L known) → M added Step 5: M found → proved!

Backward Chaining (Goal-Driven)

Start from the goal, work backwards to find facts that support it. Called goal-driven because it starts from the query and tries to prove it. Better when goal space is smaller than data space. Uses backtracking when a subgoal fails.

Query: KB ⊨ M? 1. Goal: M. Find rule with M as conclusion: B∧L→M 2. Subgoals: B and L 3. B: found in KB ✓ 4. Goal: L. Find rule: A∧P→L 5. Subgoals: A and P 6. A: found ✓; Goal: P. Rule: C∧L→P → subgoals: C and L ... L requires: A∧P→L → subgoal P again (loop!) → backtrack Alternative rule for P: D∧M→P → subgoal M → loops back to original goal! Another rule for L: A∧B→L → subgoals A and B → both found ✓ 7. Now L is proved. So B∧L→M → M is proved ✓
📌 FC vs BC Summary
Forward Chaining: All facts → fire rules → derive everything possible → check if query is among derived facts. Best when query is unknown. Backward Chaining: Start from query → find supporting rules → prove premises recursively. Best when query is specific. Both are sound and complete for Horn clause KBs.

First Order Logic (FOL / FOPL)

PL can only express facts about specific individuals. First Order Logic is more expressive — it can describe properties of objects and relationships between them, and can quantify over all or some objects using universal (∀) and existential (∃) quantifiers.

FOL Building Blocks

ConstantRefers to a specific object. E.g., Karim, AUST, 5
VariableA placeholder for any object. E.g., x, y, z. Must be bound by a quantifier.
PredicateExpresses a property or relation. E.g., Father(x,y), Greater(a,b), Subset(A,B)
FunctionMaps objects to objects. E.g., MotherOf(x), Sum(a,b), GiftFor(x)
Quantifiers∀ (for all) and ∃ (there exists). E.g., ∀x Person(x)→Mortal(x)
Ground TermA term with no variables (constants and functions of constants only). E.g., MotherOf(Karim)

Example FOL sentences:

MotherOf(Rina) = Bina ← Function returning an object ∀x ∀y ∀z (Father(x,y) ∧ Father(y,z)) → Grandfather(x,z) ← Universal rule about all objects ∃x Guest(x) ∧ ∃y Gift(y) ∧ Receives(x, y) ← There exist a guest and gift TELL(KB, { Father(Karim, Rahim), Father(Rahim, Selim), ... }) ASK(KB, ∃x ∃y Grandfather(x, y)) Returns: {x/Karim, y/Selim}, {x/Karim, y/Halim} ← substitution list

Unification in FOL

Unification is the process of making two FOL sentences identical by finding a substitution (a list of variable/term bindings). The Most General Unifier (mgu) is the substitution that makes sentences identical while leaving variables as general as possible (not over-specifying).

Unification rules: • Different predicates or constants never unify • A variable x can be unified with any term T, provided: - T does not contain x (the occurs check — prevents circular substitutions) Example: UNIFY(P1("Km", x, y), P1(z, u, F1(z))) Step 1: θ = {} Step 2: First args: "Km" vs z → θ = {z/"Km"} Apply: P1("Km", x, y) and P1("Km", u, F1("Km")) Step 3: Second args: x vs u → θ = {z/"Km", x/u} Apply: P1("Km", u, y) and P1("Km", u, F1("Km")) Step 4: Third args: y vs F1("Km") → θ = {z/"Km", x/u, y/F1("Km")} Apply: both become P1("Km", u, F1("Km")) ← Unified! mgu θ = {z/"Km", x/u, y/F1("Km")} (u stays free — most general)

Resolution in FOL

The Resolution rule is a complete inference method for FOL. Given two clauses (disjunctions of literals) that contain complementary literals, we can produce a new clause (resolvent) by applying the unifier that makes the complementary literals identical.

Resolution Rule: From: p₁ ∨ p₂ ∨ ... ∨ pᵢ (clause 1) And: q₁ ∨ q₂ ∨ ... ∨ qⱼ (clause 2) Where: pₖ and qₗ unify with mgu θ (one is positive, one is negative literal) Conclude: SUBST(θ, p₁∨...∨pₖ₋₁∨pₖ₊₁∨...∨pᵢ ∨ q₁∨...∨qₗ₋₁∨qₗ₊₁∨...∨qⱼ) (Remove the unified literals, merge the remaining literals, apply substitution)

Resolution-Refutation: To prove a query Q from KB, add ¬Q (negation of query) to KB in CNF form, then repeatedly apply Resolution. If we derive the empty clause [], a contradiction has been found, proving that Q must be true.

Converting to CNF (Conjunctive Normal Form)

Standardize variables: Rename variables in different quantifiers so they're unique

Eliminate ↔ and →: A→B becomes ¬A∨B; A↔B becomes (¬A∨B)∧(A∨¬B)

Move ¬ inward using DeMorgan's laws: ¬(A∧B) = ¬A∨¬B; ¬∀x = ∃x ¬; ¬∃x = ∀x ¬

Skolemize: Replace existential variables with Skolem functions. If ∃y (inside ∀x), replace y with GiftFor(x) — a function of the enclosing universal variables

Drop ∀ quantifiers (all remaining variables are implicitly universally quantified)

Distribute ∨ over ∧ to get a conjunction of clauses (each clause = disjunction of literals)

Example: "Every guest receives at least one gift." Original FOL: ∀x (Guest(x) → ∃y (Gift(y) ∧ Receives(x, y))) Step 1: standardize — already fine Step 2: eliminate →: ∀x (¬Guest(x) ∨ ∃y (Gift(y) ∧ Receives(x, y))) Step 3: move ¬ inward — already done Step 4: Skolemize ∃y with function GiftFor(x): ∀x (¬Guest(x) ∨ (Gift(GiftFor(x)) ∧ Receives(x, GiftFor(x)))) Step 5: drop ∀: (¬Guest(x) ∨ Gift(GiftFor(x))) ∧ (¬Guest(x) ∨ Receives(x, GiftFor(x))) Step 6: Already in CNF: Clause 1: ¬Guest(x) ∨ Gift(GiftFor(x)) Clause 2: ¬Guest(x) ∨ Receives(x, GiftFor(x)) "Karim is a guest": Clause 3: Guest(Karim) Query: "Does Karim receive a gift?" → ∃x (Gift(x) ∧ Receives(Karim, x)) Negate query → add: ¬Gift(y) ∨ ¬Receives(Karim, y) [CNF of negation] Resolve T1 (negated query) with Clause 1: θ={y/GiftFor(x)} → ¬Guest(x) ∨ ¬Receives(Karim, GiftFor(x)) Resolve with Clause 2: θ={x/Karim} → ¬Guest(Karim) Resolve with Clause 3: θ={} → [] ← Empty clause! Contradiction found. ∴ Query is PROVED: Karim receives a gift.
CH 03

Problem Solving by Informed Search

Search Problem Formulation

Many AI problems can be formulated as searching a state space for a path from an initial state to a goal state.

Initial StateStarting description of the environment. E.g., current tile arrangement in 8-puzzle.
Goal StateDesired target configuration. May be a single state or a condition to satisfy.
Successor FunctionGiven state S, returns set of (action, next_state) pairs. Defines all legal moves.
Path CostSum of step costs along a path. Uniform cost = 1 per step; or weighted by distance/time.
State SpaceSet of all states reachable from initial state via the successor function.
SolutionA path from initial state to a goal state. Optimal solution has minimum path cost.

Heuristic Functions

A heuristic function h(n) estimates the cost of the cheapest path from node n to the goal. Good heuristics guide the search toward the goal efficiently. They are derived from relaxed problems — simplified versions of the original problem.

Examples: h₁(n) = number of misplaced tiles in 8-puzzle (each tile must move ≥ once) h₂(n) = straight-line distance from city n to destination (direct route ≤ any road route) Admissibility: h(n) ≤ true cost to reach goal from n (never over-estimates) Consistency: h(n) ≤ c(n, a, n') + h(n') for every successor n' of n (triangle inequality) Every consistent heuristic is also admissible. A consistent heuristic with A* guarantees optimal solution.

Checking Admissibility and Consistency

For the road map problem with h(n) = straight-line distance to goal g:

  • Admissible: For every node n, verify h(n) ≤ shortest actual path cost from n to g. Straight-line distance is always ≤ road distance, so h₂ is admissible.
  • Consistent: For each edge (n → n') with cost c: check h(n) ≤ c(n,n') + h(n'). If h(i)=80, edge i→b costs 45, h(b)=42: 80 ≤ 45+42 = 87 ✓

Greedy Best-First Search

Selects the node that appears closest to the goal according to the heuristic: f(n) = h(n). Maintains a Priority Queue (PQ) sorted by h-values (ascending) and a Possible Path (PP) showing the current best path.

Algorithm: PQ ← {initial_node with h-value}; PP ← [] While PQ not empty: n ← node with minimum h(n) from PQ (best "looks promising") If n is goal: return PP (solution found) Mark n as visited; add to PP For each unvisited neighbor n' of n: Add n' to PQ with value h(n') Straighten PP back to root from n Properties: ✓ Fast and memory-efficient (like greedy, go to best-looking) ✗ NOT complete: can get stuck in loops or very deep paths ✗ NOT optimal: may find a solution faster but not the best one ✗ Greedy: takes locally best, not globally best
📌 PQ vs PP
PQ (Priority Queue) holds nodes to explore next, ordered by h-value. PP (Possible Path) shows which nodes are currently on the path from start to current node. When we expand a node, we straighten PP to trace the path from root to that node. PP is updated by backtracking to find the path through parent pointers in the tree.

A* Search

A* uses f(n) = g(n) + h(n) where g(n) is the actual cost from start to n and h(n) is the estimated cost from n to goal. This balances "how far we've come" with "how far we need to go."

Evaluation function: f(n) = g(n) + h(n) g(n) = actual accumulated cost from start to n h(n) = heuristic estimated cost from n to goal f(n) = estimated total cost of cheapest solution through n PQ entries: node(name, g_value, parent_index, f_value) Example: node(b, 2, 0, 87) means: - Node b - g(b) = 2 (cost to reach b from start) - Parent index 0 (came from node at index 0, which is i) - f(b) = g(b) + h(b) = 2 + 42 + ... Wait: 2 + 85 = 87 here Key properties of A*: 1. A* in TREE SEARCH is optimal if h(n) is ADMISSIBLE 2. A* in GRAPH SEARCH is optimal if h(n) is CONSISTENT 3. If h is consistent, f(n) is non-decreasing along any path 4. A* expands all nodes with f(n) < C* (optimal solution cost) 5. A* is COMPLETE (finds solution if one exists, finite nodes ≤ C*) 6. A* is OPTIMALLY EFFICIENT — no optimal algorithm expands fewer nodes
Greedy Best-First
  • f(n) = h(n) only
  • Ignores cost to reach n
  • Fast but may miss optimal path
  • Not complete, not optimal
  • Like asking "where does the goal feel closest?"
A* Search
  • f(n) = g(n) + h(n)
  • Balances actual cost + estimated remaining
  • Slower but guarantees optimal solution
  • Complete and optimal (with admissible/consistent h)
  • Like asking "what's the cheapest total route through here?"

Hill-Climbing Local Search

Hill-climbing is a local search strategy — it doesn't track paths, only the current state. It keeps moving to better neighboring states until no improvement is possible. Used when we care about the goal state itself (not how we got there), such as optimization problems.

Algorithm: current ← initial_state Loop: neighbor ← best of SUCCESSORS(current) [highest h/objective value] if h(neighbor) ≤ h(current): return current [no improvement → stop] current ← neighbor Example: 8-queens problem State: complete placement of 8 queens (one per column) h: number of attacking pairs (minimize; 0 = solution) F1: non-attacking pairs = 28 - h (maximize; max = 28) Move: shift a queen within its column to reduce h

Problems with Hill-Climbing

Local MaximumAll neighbors are worse than current state, but we're not at the global optimum. Algorithm terminates prematurely.
PlateauAll neighbors have equal value. No improvement visible, algorithm may stop even though solution is reachable.
RidgeA sequence of local maxima that's hard to navigate — standard hill-climbing zigzags inefficiently.
ShoulderLike a plateau, but slope continues upward at some point. Sideways moves can escape it.

Solutions to Hill-Climbing Problems

  • Sideways move: Allow moves to equal-value states to escape plateaus/shoulders (limit count to avoid loops)
  • Random restart: If stuck, restart from a new random state. Effective when there aren't too many local maxima. Complete with probability 1 as restarts → ∞.
  • Stochastic hill-climbing: Choose randomly among uphill moves (not always the best). Slower but explores more.
  • First-choice hill-climbing: Generate random successors one at a time, take the first one that's better. Good when many successors exist.

Genetic Algorithms

Genetic Algorithms (GA) are a stochastic beam search variant inspired by biological evolution. They maintain a population of states and evolve it toward better solutions through selection, crossover, and mutation.

Initialize Population: Create N random individuals (states). E.g., for 8-queens: {25348716, 81765432, 42561862, 31456321}

Evaluate Fitness: Apply fitness function to each individual. E.g., F = 28 - h (non-attacking pairs). Higher = better. F(a)=23, F(b)=13, F(c)=22, F(d)=17.

Check Termination: If any individual is "fit enough" (fitness ≥ threshold, say 25) → return that individual as solution.

Select Parent Generation: Keep better-performing individuals above a threshold. E.g., threshold=15, keep {a, c, d}.

Crossover: Select pairs of parents randomly. Choose a crossover point. Combine first part of parent 1 with remainder of parent 2 (and vice versa). e.g., a=253|48716 and d=314|56321 → offspring e=25356321, f=31448716

Mutation: Each gene in each offspring independently flips with small probability. e.g., 25356321 → 26356321 (digit 3 → 6). Maintains diversity.

Form New Population: Parent generation + offspring → new population. Return to step 2.

📌 Why GA Works
Crossover combines "useful blocks" (schemas) — substrings that tend to appear in high-fitness individuals. This combines the best features of multiple parents. Mutation prevents the population from converging to a local optimum by introducing random variation. Together they balance exploitation (use good solutions found) with exploration (try new areas of the state space).
CH 04

Planning Problems

Planning Domain Definition Language (PDDL)

PDDL is the standard language for specifying AI planning problems. A planning problem consists of: initial state, goal state, and a set of action schemas.

States in PDDL

  • States are conjunctions of ground, positive, function-free FOL literals
  • Closed World Assumption: anything not stated in the current state is assumed False
  • Goal states are partially specified — we only need to satisfy specified conditions, others can be anything
  • If current state contains all goal state literals → goal achieved

Action Schemas

An action schema has: Action: ActionName(param1, param2, ...) Precondition: conjunction of positive literals (conditions that must hold) Effect: conjunction of literals (positive = add, negative = delete) Example — Air Cargo: Action: Load(c, p, a) Precondition: Cargo(c) ∧ Plane(p) ∧ Airport(a) ∧ At(a,c) ∧ At(a,p) Effect: ¬At(a,c) ∧ In(p,c) [c is no longer at airport; c is now in plane] Action: Fly(p, from, to) Precondition: Plane(p) ∧ Airport(from) ∧ Airport(to) ∧ At(from,p) Effect: ¬At(from,p) ∧ At(to,p) [plane leaves 'from', arrives at 'to'] Action: Unload(c, p, a) Precondition: Cargo(c) ∧ Plane(p) ∧ Airport(a) ∧ At(a,p) ∧ In(p,c) Effect: At(a,c) ∧ ¬In(p,c) [c at airport; c not in plane anymore]

Executing an Action

  • Action can be executed if its precondition holds in current state (with appropriate variable substitutions)
  • Add list: positive literals in effect → added to current state
  • Delete list: negative literals in effect → removed from current state

Planning Strategies

Forward State Space Search
  • Start from Initial State, apply actions, reach Goal State
  • IS → action₁ → State₁ → action₂ → ... → GS
  • Challenge: branching factor can be huge (many applicable actions)
  • Uses heuristics to guide search toward goal
Backward State Space Search
  • Start from Goal State, work backwards using inverse actions
  • GS ← action_n ← ... ← IS
  • Better when goal is more constrained than initial state
  • Also uses heuristics (estimate distance to IS)
Plan types: Total Order Plan: actions must execute sequentially, no parallelism Partial Order Plan: some actions can execute in parallel (no ordering constraint between them) A planner decomposes problems into independent subproblems. Principle: Activities should be preconditioned (no action without satisfying prerequisites).

Blocks World Example

Initial State: C on D on Table; E on F on Table; A on B on Table → Clear: A, C, E Goal State: C on A on D on Table; F on E on B on Table → Clear: C, F Action schemas: Move(b1, x, b2): [move block b1 from x onto b2] Precondition: Block(b1) ∧ Block(b2) ∧ On(x,b1) ∧ Clear(b1) ∧ Clear(b2) Effect: On(b2,b1) ∧ ¬On(x,b1) ∧ Clear(x) ∧ ¬Clear(b2) MoveToTable(b1, b2): [move b1 from b2 to table] Precondition: Block(b1) ∧ Block(b2) ∧ On(b2,b1) ∧ Clear(b1) Effect: On(Table,b1) ∧ ¬On(b2,b1) ∧ Clear(b2) Plan: Action Group 1 (parallel): MoveToTable(A,B), MoveToTable(C,D) Action Group 2 (parallel): Move(A,Table,D), Move(E,F,B) Action Group 3 (parallel): Move(C,Table,A), Move(F,Table,E)
CH 05

Game Playing (Adversarial Search)

Games as Search Problems

Games are special search problems with multiple adversarial agents. In a two-player zero-sum game: what MAX wins, MIN loses (and vice versa). They plan against each other.

PlayersMAX (tries to maximize utility) and MIN (tries to minimize). Either can go first.
Utility FunctionReturns numeric value for terminal states. E.g., +1 for MAX win, -1 for loss, 0 for draw. Zero-sum: MAX's gain = MIN's loss.
Terminal TestChecks if game is over (win/draw/loss condition met).
Successor FunctionReturns (move, state) pairs. The set of legal moves from any position.
📌 Why Games are Hard
Real games have enormous state spaces (chess: ~10⁴³ legal positions). Perfect play requires considering the opponent's best response to every move. Time constraints make exhaustive search infeasible — even with modern computers, Go wasn't solved by brute force but by deep learning (AlphaGo).

MINIMAX Algorithm

MINIMAX computes the optimal strategy for MAX assuming MIN also plays optimally. It performs a complete depth-first search of the game tree.

Algorithm: MINIMAX(node, is_max_turn): if node is terminal: return UTILITY(node) if is_max_turn: return max(MINIMAX(child, False) for child in SUCCESSORS(node)) else: return min(MINIMAX(child, True) for child in SUCCESSORS(node)) Steps: 1. Generate the complete game tree 2. Find utility values at terminal nodes (leaves) 3. Propagate values upward: - At MAX levels: choose the MAXIMUM value among children - At MIN levels: choose the MINIMUM value among children 4. Choose the move at root that leads to the maximum MINIMAX value Properties: ✓ Returns optimal strategy (opener always wins if equal skill and no mistakes) ✓ Complete (if game tree is finite) ✗ Time: O(b^m) — exponential (b=branching factor, m=depth) ✗ Space: O(b*m) — linear (depth-first) ✗ Too slow for real games like chess (b≈35, m≈100)

Multi-Player Games

For games with more than 2 players, each node has a vector of utility values — one per player. Each player maximizes their own component. Alliances and coalitions naturally emerge as players cooperate to maximize their individual payoffs against third players.

Alpha-Beta Pruning

Alpha-beta pruning improves MINIMAX by eliminating branches that cannot affect the final decision — pruning subtrees that would not change the result at the root.

Alpha (α): Best value MAX can guarantee so far (starts at -∞) MAX will never choose a node with value < α Beta (β): Best value MIN can guarantee so far (starts at +∞) MIN will never choose a node with value > β Prune when: α ≥ β (the current branch can't improve either player's outcome) Algorithm sketch: ALPHABETA(node, α, β, is_max): if terminal: return UTILITY(node) if is_max: v = -∞ for child in SUCCESSORS(node): v = max(v, ALPHABETA(child, α, β, False)) α = max(α, v) if α ≥ β: break ← β-cutoff (prune remaining siblings) return v else: v = +∞ for child in SUCCESSORS(node): v = min(v, ALPHABETA(child, α, β, True)) β = min(β, v) if α ≥ β: break ← α-cutoff (prune remaining siblings) return v Improvement: With perfect ordering, reduces to O(b^(m/2)) — doubles solvable depth! In practice: good move ordering (transposition tables, killer moves) gives ~b^(3m/4)

Additional Improvements to MINIMAX

  • Cutoff depth: Instead of searching to terminal, stop at fixed depth. Replace utility with heuristic evaluation function.
  • Quiescence search: Before cutting off, continue searching past "turbulent" positions (captures, checks) until a "quiet" state is reached. Avoids horizon effect.
  • Heuristic evaluation function: Estimates utility of non-terminal state. In chess: material value (pawn=1, knight/bishop=3, rook=5, queen=9) + positional factors.
  • Transposition table: Cache previously computed positions to avoid recomputing (memoization).
  • Forward pruning: Only consider top-k moves (beam search) — reduces branching factor.
  • Opening book / Endgame database: Precomputed best moves for known positions.
CH 06

Uncertainty Management and Decision Making

Acting Under Uncertainty

Agents rarely have complete knowledge of their environment. The main tools for handling uncertainty are:

  • Probability theory — quantifies degree of belief (uncertainty about the world)
  • Utility theory — quantifies preference among outcomes (what the agent wants)
  • Decision theory = Probability + Utility — rational choice under uncertainty

Full Joint Probability Distribution

A joint probability distribution specifies the probability of every possible combination of variable values. For n binary variables, there are 2ⁿ entries.

Example: 3 Boolean variables: Toothache (T), Cavity (C), Catch (H) 8 atomic events, all probabilities sum to 1: toothache ¬toothache catch ¬catch catch ¬catch cavity: 0.108 0.012 0.072 0.008 ¬cavity: 0.016 0.064 0.144 0.576 Computing probabilities: P(cavity) = 0.108 + 0.012 + 0.072 + 0.008 = 0.2 P(toothache) = 0.108 + 0.012 + 0.016 + 0.064 = 0.2 P(cavity ∧ catch) = 0.108 + 0.072 = 0.18 [sum over toothache values] Conditional probability: P(a|b) = P(a ∧ b) / P(b) [requires P(b) > 0] P(cavity|toothache) = P(cavity ∧ toothache) / P(toothache) = (0.108 + 0.012) / (0.108 + 0.012 + 0.016 + 0.064) = 0.12 / 0.2 = 0.6

Bayes' Rule

Bayes' rule is derived from two ways of writing the product rule P(a∧b):

Product rules: P(a ∧ b) = P(a|b) × P(b) P(a ∧ b) = P(b|a) × P(a) Bayes' Rule: P(b|a) = P(a|b) × P(b) / P(a) This lets us compute the probability of a CAUSE (b) given an EFFECT (a), when we know the probability of the effect given the cause. Example: P(toothache | cavity) = 0.6 [toothache given cavity] P(cavity) = 0.1 [prior: 1 in 10 patients has cavity] P(toothache) = 0.25 [prior: 1 in 4 patients has toothache] P(cavity | toothache) = P(toothache|cavity) × P(cavity) / P(toothache) = 0.6 × 0.1 / 0.25 = 0.06 / 0.25 = 0.24 Example 2: P(Water|Coal) = 0.4, P(Coal) = 0.2, P(Water) = 0.8 P(Coal|Water) = P(Water|Coal) × P(Coal) / P(Water) = 0.4×0.2/0.8 = 0.1

Bayesian Networks

A Bayesian Network (BN) is a compact representation of a full joint probability distribution using a Directed Acyclic Graph (DAG). It dramatically reduces the number of probabilities that need to be specified.

Structure: • Nodes: random variables • Directed edges (arcs): represent direct influence / causal relationships • Each node Xᵢ has a Conditional Probability Table (CPT): P(Xᵢ | Parents(Xᵢ)) ← only depends on direct parents, not all ancestors Joint probability formula: P(x₁, x₂, ..., xₙ) = ∏ᵢ P(xᵢ | parents(Xᵢ)) Example: Cavity → Toothache, Cavity → Catch, Weather → Catch P(toothache ∧ cavity) = P(t|v) × P(v) [expand by eliminating Catch and Weather] = 0.9 × 0.002 = 0.0018 Burglary-Earthquake-Alarm example: Nodes: Burglary (B), Earthquake (E), Alarm (A), JohnCalls (J), MaryCalls (M) P(B)=0.001, P(E)=0.002; CPTs for A|B,E; J|A; M|A P(a ∧ j ∧ m ∧ ¬b ∧ e) = P(a|¬b,e) × P(j|a) × P(m|a) × P(¬b) × P(e) = 0.29 × 0.9 × 0.7 × 0.999 × 0.002 = ?
Full Joint Distribution
  • Specifies probability for every combination of variable values
  • For n binary vars: 2ⁿ entries
  • Very large, hard to specify and store
  • For 10 variables: 1024 entries
Bayesian Network
  • Only specifies P(Xᵢ | Parents(Xᵢ))
  • Far fewer parameters needed
  • Human-like: captures only direct dependencies
  • For 10 variables with ≤3 parents each: ~30 entries total

Decision Networks (Influence Diagrams)

A decision network extends a Bayesian Network with decision and utility nodes to support rational decision making.

Chance Nodes (circles)Random variables the agent is uncertain about. Have CPTs like in BN. Represent current and future states.
Decision Nodes (rectangles)Points where the agent chooses an action. Agent picks the value. Once chosen, behaves like a chance node.
Utility Node (diamond)Represents the agent's utility function. One per network. Takes all relevant variables as input.
Decision theory principle: Maximize Expected Utility (MEU) EU(A|E) = Σᵢ P(Resultᵢ(A) | E, Do(A)) × U(Resultᵢ(A)) Where: A = action being evaluated, E = evidence (current state) Resultᵢ(A) = ith possible outcome, U(S) = utility of state S Process: 1. Set evidence (observe current state variables) 2. For each possible action/decision: a. Set decision node to that action b. Compute probabilities of future state nodes using BN inference c. Compute EU = Σ P(outcome) × U(outcome) 3. Choose the action with maximum EU

Performance Metrics for Classification

The confusion matrix is the fundamental tool for evaluating classifiers.

Predicted Positive Predicted Negative Actual Positive: TP (True Pos) FN (False Neg) | P (total actual pos) Actual Negative: FP (False Pos) TN (True Neg) | N (total actual neg) Accuracy = (TP + TN) / (P + N) [overall correctness] Error rate = (FP + FN) / (P + N) = 1 - accuracy Sensitivity = TP / P [true positive rate, recall] Specificity = TN / N [true negative rate] Precision = TP / (TP + FP) [of predicted positives, how many are right?] Recall = TP / (TP + FN) = TP / P [of actual positives, how many found?] F1-score = 2 × (precision × recall) / (precision + recall) [harmonic mean] F-score = (1+β²) × precision × recall / (β² × precision + recall) [β=1: equal weight; β=2: recall weighted twice] accuracy = sensitivity × P/(P+N) + specificity × N/(P+N) Example: TP=6954, FN=46, FP=412, TN=2588; P=7000, N=3000, P+N=10000 Accuracy = (6954+2588)/10000 = 0.9542 = 95.42% Sensitivity = 6954/7000 = 99.34%; Specificity = 2588/3000 = 86.27%
📌 When to use which metric
Accuracy works when classes are balanced. When one class is rare (e.g., disease detection), accuracy is misleading — a classifier always predicting "healthy" can have 99% accuracy but 0% sensitivity. Use F1-score or sensitivity/specificity for imbalanced datasets.

Decision Tree Induction

A decision tree classifies instances by sorting them through a tree of questions about attribute values. Learned from labeled training data by recursively choosing the attribute that best separates the data by class.

Three key quantities (for attribute A, m classes): 1. Entropy (Information Content) — measures uncertainty/impurity: I(P(C₁), ..., P(Cₘ)) = -Σᵢ P(Cᵢ) × log₂(P(Cᵢ)) For our 14-sample example (9 positive, 5 negative): I(9/14, 5/14) = -(9/14)×log₂(9/14) - (5/14)×log₂(5/14) ≈ 0.940 bits Note: I = 0 (pure node, all same class), I = 1 (maximally uncertain, 50-50) 2. Remainder — expected entropy after splitting on attribute A: Remainder(A) = Σᵢ P(samples with ith value of A) × I(classes in those samples) Remainder(Age) = 5/14 × I(2/5, 3/5) + 4/14 × I(4/4, 0/4) + 5/14 × I(3/5, 2/5) = 5/14 × 0.971 + 4/14 × 0 + 5/14 × 0.971 ≈ 0.694 bits 3. Information Gain — reduction in entropy from splitting: Gain(A) = I(all samples) - Remainder(A) Gain(Age) = 0.940 - 0.694 = 0.246 bits Gains: Age=0.246, Student=0.151, Credit=0.048, Income=0.029 → Choose Age (highest gain) as root! Recursion: At each node, repeat with remaining samples and attributes. Stopping: Stop when all samples have same class (pure node), or no attributes remain, or too few samples.

The learned tree translates directly to a set of classification rules (one rule per leaf node). Each rule is a conjunction of attribute tests along the path from root to leaf.

Naïve Bayes Classifier

Naïve Bayes applies Bayes' rule for classification, with the "naïve" independence assumption: each attribute is conditionally independent of all others given the class. Despite this oversimplification, it performs surprisingly well.

Classify unknown sample X = (x₁, x₂, ..., xₙ) into class Cᵢ with highest probability: Predict: argmax P(Cᵢ|X) = argmax P(X|Cᵢ) × P(Cᵢ) [P(X) same for all classes] By the independence assumption: P(X|Cᵢ) = P(x₁|Cᵢ) × P(x₂|Cᵢ) × ... × P(xₙ|Cᵢ) = Π_k P(xₖ|Cᵢ) Estimating probabilities from training data: P(Cᵢ) = (# samples of class Cᵢ) / (total samples) For categorical attribute: P(xₖ|Cᵢ) = (# samples in Cᵢ with value xₖ) / (# samples in Cᵢ) For continuous attribute: assume Gaussian → P(xₖ|Cᵢ) = g(xₖ, μ_Ci, σ_Ci) Example: Classify X = (age=≤30, income=medium, student=yes, credit=fair) P(C₁) = 9/14 = 0.643; P(C₂) = 5/14 = 0.357 P(age=≤30|C₁) = 2/9 = 0.222; P(age=≤30|C₂) = 3/5 = 0.600 P(student=yes|C₁) = 6/9 = 0.667; P(student=yes|C₂) = 1/5 = 0.200 ... P(X|C₁) = 0.222 × 0.444 × 0.667 × 0.667 = 0.044 P(X|C₂) = 0.600 × 0.400 × 0.200 × 0.400 = 0.019 P(X|C₁)×P(C₁) = 0.044×0.643 = 0.028 ← HIGHER → predict C₁ (buys computer)

Neural Network Learning

Neural networks are loosely inspired by biological neurons. A feedforward neural network has layers of units where each unit computes a weighted sum of its inputs, passes through an activation function, and sends output to the next layer.

Network Structure

Input layer → Hidden layer(s) → Output layer Each connection has a WEIGHT wᵢⱼ (from unit i to unit j) Each unit j has a BIAS θⱼ (threshold to adjust activation) For hidden/output unit j: Net input: Iⱼ = Σᵢ Oᵢ × wᵢⱼ + θⱼ [weighted sum of inputs + bias] Output: Oⱼ = f(Iⱼ) = 1 / (1 + e^(-Iⱼ)) [sigmoid activation function] Sigmoid properties: • Maps any input to range (0, 1) • Nonlinear (crucial for learning complex patterns) • Differentiable (needed for backpropagation) • Smooth S-curve

Backpropagation Algorithm

Forward Pass: Feed input through network, compute outputs at each layer using Iⱼ = Σ Oᵢwᵢⱼ + θⱼ and Oⱼ = sigmoid(Iⱼ)

Compute Output Errors: Errⱼ = Oⱼ(1-Oⱼ)(Tⱼ - Oⱼ) where Tⱼ = target output. Oⱼ(1-Oⱼ) is the gradient of sigmoid.

Propagate Errors Backward: For hidden unit j: Errⱼ = Oⱼ(1-Oⱼ) × Σₖ Errₖ × wⱼₖ (weighted sum of errors from next layer k)

Update Weights: Δwᵢⱼ = l × Errⱼ × Oᵢ; wᵢⱼ ← wᵢⱼ + Δwᵢⱼ where l = learning rate ∈ (0,1)

Update Biases: Δθⱼ = l × Errⱼ; θⱼ ← θⱼ + Δθⱼ

Repeat for all training samples (case updating) or accumulate and update after full epoch (epoch updating)

Example computation (with network having 3 inputs, 2 hidden, 2 outputs): Input: x₁=1, x₂=0, x₃=1 Weights: w₁₄=0.2, w₁₅=-0.3, w₂₄=0.4, w₂₅=0.1, w₃₄=-0.5, w₃₅=0.2 w₄₆=-0.3, w₄₇=0.1, w₅₆=-0.2, w₅₇=0.1 Biases: θ₄=-0.4, θ₅=0.2, θ₆=0.1, θ₇=-0.1 Target: T₆=1, T₇=1 Unit 4: I₄ = 1×0.2 + 0×0.4 + 1×(-0.5) + (-0.4) = -0.7 O₄ = 1/(1+e^0.7) = 0.332 Unit 6: I₆ = 0.332×(-0.3) + 0.525×(-0.2) + 0.1 = ... ≈ -0.374 O₆ = sigmoid(-0.374) ≈ 0.408? (from slides: ~0.474) Error at 6: Err₆ = O₆(1-O₆)(T₆-O₆) = 0.474×0.526×0.526 ≈ 0.1311 Error at 4: Err₄ = O₄(1-O₄) × [Err₆×w₄₆ + Err₇×w₄₇] = 0.332×0.668 × [0.1311×(-0.3) + 0.126×0.1] ≈ -0.0059 Update w₄₆: w₄₆ = -0.3 + 0.9 × 0.1311 × 0.332 ≈ -0.261 [l=0.9]

Termination Conditions for Backpropagation

  • Weight change after an epoch is less than a threshold (converged)
  • Fixed number of epochs have been run
  • Misclassification rate on validation set stops improving

Advantages: High tolerance to noisy data; learns complex patterns. Disadvantages: Long training; topology determined empirically; poor interpretability ("black box").

Regression Analysis

Regression estimates relationships between variables — predicts a continuous output value given input features.

Linear Regression: Y = αX + β (fit a straight line to data) Least-squares solution: α = Σ(xᵢ - x̄)(yᵢ - ȳ) / Σ(xᵢ - x̄)² β = ȳ - αx̄ where x̄, ȳ are the means of x and y Error metrics: MAE (Mean Absolute Error) = (1/n) Σ |yᵢ - ŷᵢ| [less sensitive to outliers] MSE (Mean Squared Error) = (1/n) Σ (yᵢ - ŷᵢ)² [penalizes large errors more] Multiple regression: Y = β₁X₁ + β₂X₂ + β Nonlinear (polynomial): Y = β₃X³ + β₂X² + β₁X + β [transform and apply least squares] Example: X = years experience, Y = salary Given 10 data points, computed: Y = 3.5X + 23.6 Predict salary after 10 years: 3.5×10 + 23.6 = 58.6K Taka Overfitting: model fits training data too well, poor on new data (too complex) Underfitting: model too simple, can't capture underlying pattern

k-Nearest Neighbor (k-NN)

k-NN is a lazy learner — no model is built during training. Classification is done at query time by finding the k most similar training examples.

Algorithm: Given: training samples, unknown sample X, positive integer k 1. Compute distance D(X, Yᵢ) from X to every training sample Yᵢ 2. Find the k nearest neighbors (smallest distance) 3. For Classification: predict by majority vote among k neighbors 4. For Regression: predict as average value of k neighbors Distance metrics: Euclidean: D(X,Y) = √(Σᵢ(xᵢ-yᵢ)²) [most common] Manhattan: D(X,Y) = Σᵢ|xᵢ-yᵢ| [sum of absolute differences] Minkowski: D(X,Y) = (Σᵢ|xᵢ-yᵢ|^p)^(1/p) [general; p=1→Manhattan, p=2→Euclidean] Hamming distance: for categorical attributes (number of differing positions) Properties: ✓ Simple, no training phase, handles multiclass naturally ✓ Non-parametric — no assumptions about data distribution ✗ Slow at query time O(n×d) per prediction ✗ Sensitive to irrelevant features and scale of attributes ✗ Majority vote can fail with skewed class distributions Key concern: choosing k. Small k = sensitive to noise; large k = smoother but may miss local structure.

k-Means Clustering

k-means is an unsupervised learning algorithm that partitions n objects into k clusters, minimizing within-cluster variance (distance to centroid).

Initialize: Choose k initial centroids randomly from data points (or use k-means++ for better initialization)

Assignment: Assign each data point to the nearest centroid (using Euclidean distance)

Update: Recompute each centroid as the mean of all points assigned to it

Repeat steps 2–3 until: no point changes cluster, OR centroids stabilize, OR maximum iterations reached

Objective (minimize): E = Σᵢ₌₁ᵏ Σₚ∈Cᵢ |p - mᵢ|² [sum of squared distances to cluster centroid] where mᵢ = mean position (centroid) of cluster Cᵢ Applications: customer segmentation, image compression, anomaly detection, document clustering, motion analysis Choosing k: Elbow method (plot E vs k, choose "elbow" point), Silhouette method (measure cohesion vs separation) Limitation: NP-hard in general; heuristic k-means may find local minimum Results depend on initial centroid choice

Support Vector Machines (SVM)

SVMs find the maximum margin hyperplane — the separating surface that is as far as possible from all training samples. The goal is to find the hyperplane that maximizes the gap between classes.

Hyperplane: W·X + b = 0 [W = weight vector, b = bias scalar] For binary classification (+1 / -1 classes), adjust W and b so that: H1: W·X + b ≥ +1 for class +1 samples H2: W·X + b ≤ -1 for class -1 samples Margin = 2/||W|| where ||W|| = √(w₁² + w₂² + ... + wₙ²) Maximize margin → minimize ||W|| → optimization problem (convex quadratic programming) Support Vectors: training samples that lie ON hyperplanes H1 or H2 (the closest points to the separating hyperplane — they "support" it) Key intuition: Support vectors are the most important training points. The position of the hyperplane only depends on these few samples. Kernel Trick: For non-linearly separable data, map to higher-dimensional space where data becomes linearly separable. Common kernels: • Linear: K(X,Y) = X·Y • Polynomial: K(X,Y) = (X·Y + 1)^d • RBF/Gaussian: K(X,Y) = exp(-||X-Y||²/2σ²) [most widely used] Advantages: Resistant to overfitting; works well in high dimensions; powerful with kernels Applications: handwriting recognition, face detection, protein classification, text categorization
